# DCollections

**DCollections** is a lightweight, high-performance collection library for Delphi, designed with a different goal than the standard RTL collections.

Instead of focusing only on features, DCollections was built to **reduce generic code bloat**, **decrease compilation time**, and **minimize generated DCU size** while providing a modern and expressive API.

The library centralizes collection algorithms in non-generic base classes, leaving generic classes as thin typed facades. This significantly reduces the amount of code generated by the Delphi compiler when projects contain hundreds or thousands of generic specializations.

## Key Features

- 🚀 Optimized for large Delphi projects
- 📦 Significantly reduces generic code generation
- ⚡ Faster project compilation
- 💾 Smaller DCU files
- 🧩 Modern collection API
- 🔒 Type-safe generic wrappers
- ♻️ Centralized non-generic algorithms
- 📚 Compatible with Delphi 10.3 Rio and newer
- ❌ No external dependencies
- ❌ No RTTI requirements
- ❌ No Spring4D dependency

## Philosophy

Traditional Delphi generic collections generate a complete implementation for every specialized type.

DCollections follows a different approach:

- Generic classes contain only the strongly-typed public API.
- Collection algorithms live in shared non-generic base classes.
- Memory management and collection logic are reused across all specializations.
- The compiler generates substantially less duplicated code.

The result is a collection library particularly suited for enterprise applications with hundreds of domain classes.

## Current Collections

- `TDObjectList<T>`

More collections are planned:

- `TDList<T>`
- `TDDictionary<TKey, TValue>`
- `TDQueue<T>`
- `TDStack<T>`

## Design Goals

- Reduce generic code bloat
- Improve compilation speed
- Reduce generated DCU size
- Keep runtime performance competitive
- Provide a clean and intuitive API
- Minimize generic specialization
- Preserve Delphi coding style

# Usage

## Creating a typed object list

```delphi
type
  TCustomer = class
  private
    FId: Integer;
    FName: string;
  public
    property Id: Integer read FId write FId;
    property Name: string read FName write FName;
  end;

var
  Customers: TDObjectList<TCustomer>;
begin
  Customers := TDObjectList<TCustomer>.Create(True);
  try

  finally
    Customers.Free;
  end;
end;
```

---

## Adding items

```delphi
Customers.Add(TCustomer.Create);
Customers.Add(TCustomer.Create);
Customers.Add(TCustomer.Create);
```

---

## Iterating

```delphi
var
  Customer: TCustomer;

for Customer in Customers do
  Writeln(Customer.Name);
```

---

## Finding the first matching item

```delphi
var
  Customer: TCustomer;

Customer :=
  Customers.First(
    function(const Item: TCustomer): Boolean
    begin
      Result := Item.Id = 100;
    end);
```

---

## Finding the last matching item

```delphi
Customer :=
  Customers.Last(
    function(const Item: TCustomer): Boolean
    begin
      Result := Item.Name = 'John';
    end);
```

---

## Finding the index

```delphi
var
  Index: Integer;

Index :=
  Customers.FindIndex(
    function(const Item: TCustomer): Boolean
    begin
      Result := Item.Id = 100;
    end);
```

---

## Checking if an item exists

```delphi
if Customers.Any(
  function(const Item: TCustomer): Boolean
  begin
    Result := Item.Id = 100;
  end)
then
  ShowMessage('Customer found.');
```

---

## Counting matching items

```delphi
ActiveCount :=
  Customers.CountWhere(
    function(const Item: TCustomer): Boolean
    begin
      Result := Item.Active;
    end);
```

---

## Removing matching items

```delphi
Customers.RemoveWhere(
  function(const Item: TCustomer): Boolean
  begin
    Result := Item.Inactive;
  end);
```
---

## Extracting an object

```delphi
Customer := Customers.Extract(Customer);

try
  // use Customer
finally
  Customer.Free;
end;
```

---

## Ownership

```delphi
// Owns objects
Customers := TDObjectList<TCustomer>.Create(True);

// Does not own objects
Customers := TDObjectList<TCustomer>.Create(False);
```

---

## Bulk insert

```delphi
Customers.AddRange([
  Customer1,
  Customer2,
  Customer3
]);
```

---

## Access by index

```delphi
Customer := Customers[0];

Customers[0].Name := 'Peter';
```

---

## Clear

```delphi
Customers.Clear;
```

## Performance

DCollections is not intended to outperform hand-written loops.

Its primary goal is to reduce the amount of code generated by the compiler while maintaining competitive runtime performance.

In internal benchmarks with hundreds of specialized collections, DCollections produced significantly smaller DCU files and substantially reduced build times compared to the standard `Generics.Collections` implementation.
